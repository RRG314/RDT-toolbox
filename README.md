# RDT Toolbox Showcase

This repository is the consolidation point for your corrected RDT work.

It includes:
- base RDT variants
- optimized RDT variants (only kept when benchmarked improvements are measurable)
- reproducible tests and benchmark reports
- explicit failure cases (where RDT is worse)

The goal is practical honesty: make useful RDT tools people can run, verify, and compare.

## Repository Structure

- `src/rdt_showcase/`
  - `rdt_index.py`: RDT hierarchy index and shard assignment primitives
  - `baselines.py`: hash map / sorted array / B-tree-like baselines
- `tools/`
  - `benchmark_rdt_index.py`: full W1/W2/W3/W4 benchmark suite
  - `run_showcase.py`: cross-repo benchmark aggregator (`rdt-noise`, `rdt-spatial-index`, `rdt256`)
  - `run_tests.py`: unit test runner
- `tests/`: independent correctness tests
- `results/`: JSON artifacts
- `docs/`: human-readable benchmark reports

## One-Command Workflows

## Benchmark Scope

The RDT index benchmark runs all required workloads:
- W1 uniform random integer keys
- W2 sequential keys (timestamp-like)
- W3 clustered keys
- W4 mixed read/write (`80/20`, `95/5`)

It measures:
- build time and memory
- lookup latency (`p50`, `p95`)
- bucket/range query latency
- partition balance and worst bucket size
- shard movement when shard counts change

It includes adversarial checks and reruns with a minimal RDT-based fix when depth-only RDT loses.

## Current Packaging Choice

Based on measured win cases, this repo packages **Option B: RDTShard**:
- deterministic shard assignment from RDT hierarchy
- stable remapping via ancestor buckets + jump-consistent hash
- explicit comparison vs standard modulo partitioning
